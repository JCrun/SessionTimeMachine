# SessionTimeMachine 项目计划

目标：在 Sublime Text 3 中实现 Session 实时备份、历史回滚与搜索能力，覆盖配置文件与 `.sublime_session` 的快照、索引与恢复流程。

## 范围与原则
- 以 ST3（Python 3.3）为运行环境，避免重型依赖。
- 本地快照优先，远端同步为可选扩展。
- 所有 I/O 走异步，避免阻塞 UI 线程。

## 里程碑

### 阶段 1：基础骨架与本地快照
交付物
- 事件监听：保存 `Packages/User` 下配置时触发备份。
- 定时任务：周期性备份 `.sublime_session`。
- 本地存储结构：`.sync_backup/session` 与 `.sync_backup/user`。
- 可配置项：快照间隔、保留数量、备份目录。

验收标准
- 关闭/切换窗口后，`session` 目录持续生成时间戳快照。
- 保存任意 User 配置文件后，`user` 目录产生对应备份。
- 超出保留数量会自动清理最旧快照。

### 阶段 2：远端同步（可选）
交付物
- Git 或 WebDAV 方案二选一的同步适配层。
- 启动时拉取、快照后推送的异步流程。

验收标准
- 远端与本地快照集合一致，支持冲突检测与提示。

### 阶段 3：Session 解析与回滚
交付物
- 解析 `.sublime_session` JSON 的解析器。
- 回滚入口命令 + Quick Panel 选择时间点。
- 文件路径映射规则（跨平台路径转换）。

验收标准
- 能重建历史窗口与标签页；未保存内容可回填。

### 阶段 4：历史索引与搜索
交付物
- SQLite/FTS 索引未保存文本。
- 搜索面板 + Quick Panel 预览与恢复。

验收标准
- 关键词检索能够定位并恢复历史草稿。

## 风险与对策
- Session 文件覆盖：只能解析后用 API 重建，不直接替换。
- 跨平台路径：配置化映射表与规则化替换。
- 性能风险：限制快照频率 + 异步任务 + 过期清理。
